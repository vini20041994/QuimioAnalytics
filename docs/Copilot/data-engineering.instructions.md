## Engenharia de Dados

Sempre:

- validar schemas;
- validar tipos;
- validar ranges;
- validar unidades;
- detectar inconsistências;
- criar pipelines idempotentes;
- criar logs de transformação;
- registrar origem dos dados;
- manter lineage;
- evitar perda silenciosa.

Toda transformação deve:

- ser rastreável;
- ser reproduzível;
- possuir logs;
- possuir validação;
- possuir rollback quando possível.
