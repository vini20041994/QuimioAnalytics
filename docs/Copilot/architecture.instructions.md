## Arquitetura

Sempre:

- utilizar arquitetura modular;
- separar domínio, aplicação, infraestrutura e apresentação;
- evitar acoplamento forte;
- evitar regras de negócio em controllers;
- criar serviços reutilizáveis;
- utilizar DTOs e validações explícitas;
- implementar versionamento de APIs;
- criar logs estruturados;
- utilizar princípios SOLID;
- priorizar Clean Architecture;
- priorizar Clean Code;
- utilizar convenções consistentes;
- criar componentes reutilizáveis;
- manter tipagem forte.
